## 概要

この差分では、Explorer のツリー表示まわりに対して、主に以下の機能が追加されています。

- テーブル一覧に限定して絞り込める **Table Filter** 機能
- ツリーフィルタの正規表現検索対応
- フィルタ中に未ロードのツリーノードを自動ロードし、非同期ロード完了後も絞り込みを再適用する仕組み
- テーブル／ビュー選択時に、列情報とインデックス情報を結果ペインへ表示するショートカット
- MySQL / MariaDB 向けの列・インデックス詳細表示強化
- キーマップ、状態判定、フッター表示、入力コンテキスト、型プロトコルの拡張

## Explorer の Table Filter 機能

### 新しいショートカット

Explorer の tree モードに `t` キーが追加され、`table_filter` アクションを起動できるようになりました。

このフィルタは通常のツリーフィルタとは異なり、選択中のデータベース配下にある `Tables` フォルダをスコープとして扱い、テーブルノードだけを検索対象にします。

### 起動できる場所

`table_filter` は以下のノード上で利用できます。

- データベースノード
- `Tables` フォルダノード
- テーブルノード

一方で、`Views` など他のフォルダや、接続フォルダなどテーブル検索の文脈ではない場所では起動できないように制御されています。

### 入力コンテキストの拡張

状態判定のために、現在選択中のツリーノードがフォルダの場合、その `folder_type` を `InputContext` に含めるようになりました。

これにより、同じ `folder` ノードでも `tables` フォルダだけに Table Filter を許可する、といった細かい判定が可能になっています。

### フッター表示

Table Filter が利用可能な状態では、フッターの操作ヒントにも `Table Filter` が表示されるようになりました。

対象は以下です。

- データベースノード上
- `Tables` フォルダ上
- テーブルノード上

## ツリーフィルタの検索機能強化

### 正規表現検索

ツリーフィルタで正規表現検索ができるようになりました。

以下のプレフィックスが正規表現モードとして扱われます。

- `/pattern`
- `re:pattern`
- `r:pattern`

正規表現は大文字・小文字を区別せずにコンパイルされ、マッチした範囲がハイライトされます。

不正な正規表現が入力された場合は例外を外に出さず、エラー情報を内部状態に保持し、マッチなしとして扱うようになっています。

### 通常フィルタと Table Filter のスコープ分離

通常の Explorer フィルタは、接続・接続フォルダ・データベースなどの上位階層を主な検索対象として扱います。

一方で Table Filter は、`Tables` フォルダ配下のテーブルを対象とするスコープ付きフィルタとして動作します。

そのため、テーブル検索だけをしたい場合に、接続名・データベース名・その他オブジェクトに検索結果が混ざりにくくなっています。

### フィルタ中のカーソル移動

Table Filter では、検索結果の次・前へ移動したときに、該当テーブルへツリーカーソルを移動します。

通常のフィルタでは従来通り、該当ノードの祖先を展開して選択する動作を維持しています。

### フィルタ確定後の挙動

通常のツリーフィルタは Enter で確定するとフィルタを閉じ、ツリーを元に戻して選択ノードをアクティブ化します。

Table Filter の場合は、テーブル一覧を絞り込んだ状態を残しつつ、選択したテーブルへカーソルを置く挙動になっています。

また、ツリー再構築や非同期ロードをまたいでもカーソル位置を復元できるよう、選択ノードのパスを記憶する処理が追加されています。

## 非同期ロードとフィルタの連携

### 未ロードフォルダのロード開始

Table Filter 中に検索対象の `Tables` フォルダなどがまだロードされていない場合、フィルタ処理側からロードを開始します。

ロード対象になり得るフォルダとして、以下が定義されています。

- databases
- tables
- views
- indexes
- triggers
- sequences
- procedures

Table Filter の主用途はテーブル検索ですが、階層構造やデータベース配下のロード状況に対応するため、関連するロード可能フォルダを扱える設計になっています。

### ロード中ノードの表示維持

検索条件に合わないノードを非表示にする処理の中で、ロード中のフォルダやローディングプレースホルダは消さずに残すようになりました。

これにより、検索中にロードが走っているにもかかわらず `Loading` ノードが消えてしまう問題を避けています。

### ロード完了後の再フィルタ

列情報やフォルダ内容の非同期ロードが完了した後、ツリーフィルタが表示中であれば自動的にフィルタを再適用します。

そのため、フィルタ入力後に遅れて読み込まれたテーブルやオブジェクトも、検索結果に反映されるようになっています。

## テーブル列情報・インデックス情報表示

### 新しいショートカット

Explorer の tree モードに以下のショートカットが追加されました。

- `c`: `show_table_columns`
- `i`: `show_table_indexes`

テーブルまたはビューを選択しているときに利用できます。

### Show Columns

`show_table_columns` では、選択中のテーブル／ビューのカラム一覧を結果ペインに表示します。

通常プロバイダではスキーマサービスの `list_columns` を利用し、以下の列を表示します。

- 行番号
- カラム名
- データ型
- 主キーかどうか

表示後は、最後の結果列・結果行・行数も更新されます。

### MySQL / MariaDB の Show Columns

MySQL / MariaDB では、通常のスキーマサービスではなく `SHOW FULL COLUMNS FROM ...` を実行し、その結果をそのまま結果ペインに表示します。

これにより、MySQL 固有の詳細情報も確認しやすくなっています。

### Show Indexes

`show_table_indexes` では、選択中のテーブルに紐づくインデックス一覧を結果ペインに表示します。

通常プロバイダでは `list_folder_items("indexes", database)` から取得したインデックス一覧を、選択中テーブル名と照合して絞り込みます。

その後、各インデックスについて可能であれば `get_index_definition` を呼び出し、以下の情報を表示します。

- インデックス名
- 対象カラム
- ユニークかどうか
- 定義文

スキーマ名付きのテーブル名にも対応するため、インデックス側のテーブル名と選択中テーブル名を比較する補助処理も追加されています。

### MySQL / MariaDB の Show Indexes

MySQL / MariaDB では `SHOW INDEX FROM ...` を実行し、その結果をそのまま結果ペインに表示します。

また、MySQL 系プロバイダのインデックス一覧取得では、`PRIMARY` インデックスを除外しないように変更されました。

これにより、主キーインデックスも Explorer 側のインデックス情報として扱えるようになっています。

## キーマップ・状態管理の変更

### キーマップ追加

デフォルトキーマップに以下が追加されました。

- `c` → `show_table_columns`
- `i` → `show_table_indexes`
- `t` → `table_filter`

### キーバインディングスナップショットの型安全化

キーバインディングのデバッグイベント出力では、内部メソッドの存在確認を `hasattr` から `getattr` + `callable` ベースに整理しています。

また、内部フラグ更新時に `typing.cast` を使うことで、型チェッカー向けの `type: ignore` を減らしています。

## MySQL 接続プロバイダの変更

### 型注釈の整理

`apply_database_override` の型注釈で文字列による前方参照をやめ、直接 `ConnectionConfig` を指定する形に整理されています。

### インデックス一覧の変更

MySQL / MariaDB の `information_schema.statistics` 参照時に、`index_name != 'PRIMARY'` の条件が削除されました。

これにより、主キーインデックスも一覧に含まれるようになっています。

## ツリー再読み込み・カーソル復元

データベース一覧やテーブル一覧などのフォルダロード完了後に、以下の処理が強化されています。

- 展開済みパスの復元
- 展開済みノードの再ロード
- 保留中カーソル位置の復元
- フィルタが開いている場合の再フィルタ

これにより、フィルタ中や非同期ロード中にツリー構造が更新されても、表示状態やカーソル位置が崩れにくくなっています。

## 型プロトコルの拡張

Explorer 関連の Protocol に、追加された状態値やメソッドが反映されています。

追加された主な状態値は以下です。

- 正規表現フィルタ状態
- フィルタ適用済みフラグ
- Table Filter のスコープパス
- ロード可能フォルダ種別

追加された主な操作は以下です。

- `action_table_filter`
- フィルタセッション開始・終了処理
- Table Filter の検索ルート解決
- 正規表現検索処理
- フィルタ対象ノード判定
- フィルタ中ロード処理
- カーソル復元処理

## 今回の整理で行った重複削減

差分内で追加されていた処理のうち、同じ初期化や表示更新を複数箇所で繰り返していた部分を整理しました。

具体的には、以下を共通化しています。

- 通常ツリーフィルタと Table Filter の開始時に行う一時状態のリセット処理
- テーブル列情報・インデックス情報を結果ペインへ反映し、最後の結果状態を更新する処理
- MySQL / MariaDB の `SHOW ...` 系メタデータクエリで、カーソル実行・行取得・カーソルクローズを行う処理

これにより、今後フィルタやメタデータ表示の挙動を変更するときに、同じ修正を複数箇所へ入れる必要が減っています。
